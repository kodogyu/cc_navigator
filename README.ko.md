# cc_navigator

[English](README.md) · **한국어**

> 실행 중인 모든 Claude Code·Codex 세션을 한 줄씩 보여주고, 내 입력을 기다리는 세션을
> 강조하며, 그 세션의 터미널로 **바로 이동**하거나 **답장을 입력**할 수 있게 해 주는
> 항상 위에 떠 있는 패널 — 알림이 어느 세션에서 왔는지 창을 뒤질 필요가 없습니다.

## 왜 필요한가

여러 개의 Claude Code·Codex 세션을 여러 터미널 창에 띄워 두고 씁니다. 세션 하나가 입력을
기다릴 때 데스크톱 알림이 뜨지만, *어느* 세션인지 알려주지 않고 그 세션에 빠르게
도달할 방법도 없습니다.

cc_navigator는 세션마다 **한 줄, 상태 아이콘 하나, 클릭 한 번으로 이동**을 줍니다.
누가 내 입력을 기다리는지 표시하고, 클릭하면 그 세션의 터미널을 앞으로 띄우고,
tmux 페인으로 한 줄 답장을 보내고, 세션이 *내 차례*가 되면 그 세션을 이름으로 알려
주는 데스크톱 알림을 띄웁니다.

## 설치 & 실행

**X11 + GNOME + tmux** 전용 도구입니다. 서드파티 파이썬 의존성 없이 표준 라이브러리와
시스템 `gi` 바인딩만 씁니다.

| | |
|---|---|
| 디스플레이 서버 | **X11** (Wayland 아님) |
| 데스크톱 | **GNOME Shell + `Eval` 잠금 해제** (GNOME 41부터 막힘, 3.36.9에서 개발). 없어도 앱은 동작하며 이동 버튼만 비활성화됩니다. |
| 멀티플렉서 | **tmux ≥ 3.0** — 프로젝트마다 세션 하나, 각자 자기 터미널 창에 |
| 인터프리터 | **`/usr/bin/python3` ≥ 3.8 + PyGObject** — `apt install python3-gi python3-gi-cairo gir1.2-gtk-3.0` |
| 추가 필요 | `gdbus`, `xprop`, `notify-send`(libnotify) |

```sh
git clone https://github.com/kodogyu/cc_navigator.git
cd cc_navigator

./bin/cc-navigator-doctor      # 사전 요건 점검 후 고칠 점을 정확히 알려줌
./install                      # 선택: cc-navigator를 PATH에 심볼릭 링크
cc-navigator &                 # 또는 ./bin/cc-navigator &
```

그다음 **세션이 나타나게 하려면** 훅을 한 번 켭니다. **설정**(⚙ 톱니바퀴) → **통합**에서
사용하는 도구의 훅을 켜세요.

- **Claude Code 훅 설정** — `~/.claude/settings.json`에 병합합니다.
- **Codex 훅 설정** — `$CODEX_HOME/hooks.json`(기본 `~/.codex/hooks.json`)에 병합합니다.
  설치 후 Codex에서 `/hooks`를 열어 `cc-navigator-hook`을 최초 한 번 신뢰해야 합니다.

이후 시작하거나 재개하는 세션이 목록에 뜹니다. Codex 행에는 파란 **Codex** 배지가 붙습니다.
현재 Codex는 첫 프롬프트까지 최초 라이프사이클 훅을 미루므로, 패널이 같은 사용자의 tmux
소켓에서 실제 Codex 프로세스를 찾아 차분한 임시 행을 즉시 표시합니다. 첫 훅이 오면 이 행은
완전한 실시간 상태로 교체됩니다.

**이동 기능은 tmux가 창 제목을 소유해야 동작합니다** — 패널이 창을 찾는 유일한 주소입니다.
`~/.tmux.conf`에 아래를 넣으세요(닥터가 검사합니다):

```tmux
set -g set-titles on
set -g set-titles-string 'ccnav:#{session_name}'
```

> **닥터를 먼저 실행하세요.** 무엇이 빠졌는지 알려줄 뿐 아니라, tmux 3.0a의 설정
> 함정 — `-g`/`-q`/`-s` 없는 `set` 계열 한 줄이 서버를 조용히 망가뜨려 공백을 보내는
> 순간 모든 세션을 죽일 수 있음 — 을 임시 소켓에서 재현해, 실제 tmux를 건드리기 전에
> 알려줍니다.

**업데이트:** 설정 ⚙ → **업데이트 확인** 이 이 체크아웃을 최신 `master`로 fast-forward
하고 재시작합니다(로컬 변경이 있으면 중단). 각 버전이 무엇을 바꾸는지 — 여러분의 머신에서
무엇을 새로 하기 시작하는지 포함 — 는 [CHANGELOG.md](CHANGELOG.md)에 있습니다.

## 화면 구성

각 세션은 **한 줄**입니다. 상태 아이콘, 제목, 그리고 (선택했을 때) 작업 디렉터리,
마지막 프롬프트, 답장 입력칸, 이동 버튼이 나옵니다.

- 🔴 **빨강** — 내 입력을 기다림(권한·질문·계획). 제목 표시줄 배지가 개수를 셉니다.
- 🟢 **초록** — 턴을 끝내고 대기 중. 클릭하면 "봤음"(초록 체크 ✓)으로 표시됩니다.
- ↻ **회전** — 에이전트가 작업 중. **서브에이전트**를 띄운 작업 중 세션은 뒤에 두 번째
  스피너가 도는 **파란 점**으로 보입니다.

상단 **정렬** 드롭다운으로 보기를 전환합니다.

- **상태별 정렬** — 우선순위 순서의 **접을 수 있는** 네 섹션: 입력이 필요한 세션 →
  작업 중 → 보고 완료 → 확인 완료(✓).
- **그룹별 정렬** — 프로젝트 디렉터리마다 폴더 하나, 각자 카운트(빨강 ● → ↻ →
  초록 ● → ✓)를 가짐. 헤더를 드래그해 그룹 순서를 바꾸고, 행의 ⠿ 손잡이로 세션을
  옮기고, 연필로 이름을 바꾸고, **자동 정렬**로 다시 그룹핑합니다.

**세션 조작:** 행을 클릭하고, 한 줄 입력 후 **Enter**로 전송하거나, **"세션으로 이동"**
을 눌러 그 터미널을 앞으로 띄웁니다.

**치워 두기:** 좌상단 쉐브론으로 패널을 제목 표시줄까지 접고, **길게 누르면** 화면
가장자리에 얇은 바로 붙일 수 있습니다(가장자리를 따라 이동, 떼어내기로 복원).

**놓치지 않기:** 세션이 *내 차례*(입력 필요 또는 작업 완료)가 되면 데스크톱 알림이
뜹니다. 기본 켜짐이며, 알림·불투명도·색·글꼴 등은 **설정**에서 조절합니다.

**사용량 확인:** 창 오른쪽 아래 **"사용량 확인"** 버튼을 누르면 Claude Code와 Codex를
각각 나눠 플랜과 한도 — 세션, 일간/주간, 모델별 한도 — 를 막대·퍼센트·리셋 시각과 함께
보여줍니다. 한쪽 계정이 없거나 실패해도 다른 쪽 결과는 그대로 표시됩니다.

- Claude Code는 `~/.claude/.credentials.json`의 OAuth 토큰으로 Anthropic의 **비공식 내부**
  `/api/oauth/usage` 엔드포인트를 호출하므로 업데이트로 동작을 멈출 수 있습니다.
- Codex는 로컬 `codex app-server`의 `account/rateLimits/read`를 사용합니다. cc_navigator가
  Codex 인증 토큰을 직접 읽거나 전송하지 않습니다.
