# cc_navigator

[English](README.md) · **한국어**

> 살아 있는 모든 Claude Code 세션을 한 줄씩 나열하고, 입력을 기다리는 세션을 강조
> 표시하며, 그 세션의 터미널로 **바로 이동**하거나 **답장을 곧장 입력**할 수 있는
> 항상 위에 떠 있는 패널입니다. 알림이 어느 세션에서 왔는지 찾으려고 수십 개의 창과
> 탭을 뒤질 필요가 없습니다.

## 소개

여러 개의 Claude Code 세션을 여러 터미널 창에 걸쳐 동시에 돌립니다. 그중 하나가
입력을 필요로 하면 데스크톱 알림이 뜨지만, *어느* 세션인지는 알려주지 않고, 그
세션에 닿으려면 창과 탭을 일일이 뒤지는 수밖에 없습니다.

cc_navigator는 모든 세션에 **한 줄, 하나의 상태 표시, 그리고 그곳으로 가는 한 번의
클릭**을 줍니다.

- **상태를 한눈에** — 입력을 기다리는 세션은 빨간 점 ●, 턴을 끝낸 세션은 초록 점 ●,
  Claude가 작업 중이면 회전하는 화살표 ↻. subagent를 띄운 세션은 main 아이콘 뒤에
  두 번째 spinner가 겹쳐서 — 내 입력을 기다리면서(빨강) 도우미는 도는 상황도 한눈에
  보입니다. 초록 점을 클릭하면 "봤음"(빈 박스)으로 표시됩니다. 제목 표시줄의 배지가
  입력이 필요한 세션 수를 셉니다.
- **두 가지 보기** — **상태별**(입력 필요 / 보고 완료 / 작업 중 섹션) 또는
  **그룹별**(작업 디렉터리마다 폴더 하나)로 정렬합니다.
- **이동** — *"해당 세션으로 이동"*을 클릭하면 그 세션의 터미널 창이 앞으로 옵니다.
- **답장** — 행에 한 줄을 입력하고 Enter를 누르면 그 세션의 tmux 페인으로 곧장
  전달됩니다.
- **접기 & 붙이기** — 패널을 제목 표시줄만 남기고 접거나, 화면 가장자리에 딱 붙는
  얇은 바로 도킹할 수 있습니다(가장자리를 따라 이동도 가능).
- **그룹 관리** — 세션 순서를 바꾸거나, 그룹 사이로 옮기거나, 그룹 이름을 바꾸거나,
  버튼 하나로 디렉터리 기준 자동 그룹화를 할 수 있습니다.

### 동작 방식

Claude Code는 세션 생명주기 이벤트마다 **훅(hook)** 을 실행합니다. 작은 shim
(`bin/cc-navigator-hook`)이 각 이벤트를 세션별 상태 파일로 기록합니다. 패널은 그
상태 파일들을 살아 있는 `tmux` 페인과 조인해서 무엇을 보여줄지 결정합니다.

- **생존 여부는 통보받는 게 아니라 파생됩니다.** `tmux`에서 사라진 세션은 다음 갱신
  때 패널에서 사라지고 그 상태 파일은 정리되므로, 죽은 세션이 유령 행을 남기지
  않습니다.
- **이동은 창을 제목으로 지정합니다.** `tmux`의 `set-titles-string`이 외부 X11 창
  제목에 `ccnav:<세션>`을 새기고, GNOME Shell이 그 제목을 가진 창을 활성화합니다.
- **답장은 한 줄을 주입합니다.** 세션의 페인에 `tmux send-keys -l`로 보내므로,
  텍스트가 문자 그대로 전달되고 셸이 절대 해석하지 않습니다.

모든 모듈과 테스트를 관통하는 하나의 규칙:

> **API의 자기 보고를 믿지 마라. 한 채널로 실행하고, 다른 채널로 검증하라.**
> (`gdbus`는 GNOME `Eval`이 아무것도 하지 않아도 종료 코드 `0`을 내므로, 이동
> 경로는 `Eval`로 *실행*하고 그 효과를 `xprop`으로 *검증*합니다.)

## 요구 사항

의도적으로 좁게 설계된 **X11 + GNOME + tmux** 도구입니다. 서드파티 파이썬 의존성은
없습니다 — 표준 라이브러리와 시스템 `gi` 바인딩뿐입니다.

| | |
|---|---|
| 디스플레이 서버 | **X11** (Wayland 아님) — 포커스를 `xprop`으로 `_NET_ACTIVE_WINDOW`를 읽어 검증 |
| 데스크톱 | **`Eval`이 잠기지 않은 GNOME Shell** — GNOME 41부터 차단됨. 3.36.9에서 개발 |
| 터미널 멀티플렉서 | **tmux ≥ 3.0** — 세션을 tmux 페인으로 지정 |
| 인터프리터 | **PyGObject가 있는 `/usr/bin/python3` ≥ 3.8** (`apt install python3-gi gir1.2-gtk-3.0`) |
| 추가 필요 | `gdbus`, `xprop` |

GNOME `Eval`을 쓸 수 없어도 **앱은 여전히 실행됩니다** — 이동 버튼이 비활성화되고
상태 표시줄이 이유를 설명합니다. 세션 나열과 답장 입력은 그것 없이도 동작합니다.

## 설치

```sh
git clone https://github.com/kodogyu/cc_navigator.git
cd cc_navigator

./run-tests                    # 선택: "Ran 445 tests / OK" 기대
./bin/cc-navigator-doctor      # 환경을 점검하고 무엇을 고쳐야 하는지 정확히 알려줌
```

**1. 먼저 doctor를 실행하세요.** 설정 파일을 보고 추측하지 않습니다 — 아래 경고에
나오는 딱 하나의 tmux 오류를 임시 소켓에서 재현해 보고, `~/.tmux.conf`에 어떤 줄을
추가해야 하는지 정확히 알려줍니다.

**2. 런처를 PATH에 올립니다**(선택이지만 편리함):

```sh
./install        # bin/cc-navigator를 ~/.local/bin에 심볼릭 링크 (멱등)
```

`install` 스크립트는 `cc-navigator`를 PATH에 추가하기만 합니다. Claude Code 설정은
일부러 **건드리지 않습니다** — 훅 연결은 앱 안의 토글(4단계)이라, 직접 보고 되돌릴
수 있습니다.

**3. 패널을 실행합니다:**

```sh
cc-navigator &          # ./install을 건너뛰었다면 ./bin/cc-navigator &
```

**4. Claude Code 훅을 켭니다 — 이것이 세션을 나타나게 하는 핵심 단계입니다.** 패널의
**설정**(⚙ 톱니바퀴 버튼)을 열고, **통합** 항목에서 **"Claude Code 훅 설정"** 을
켜세요. 훅 shim을 `~/.claude/settings.json`에 알아서 병합해 줍니다(`SessionStart`,
`UserPromptSubmit`, `Notification`, `Stop`, `SessionEnd`, `PreToolUse`,
`PostToolUse`, `SubagentStart`, `SubagentStop`). 같은 화면에서 앱 목록 등록(**"앱 목록에 등록"**)과
로그인 시 자동 실행(**"로그인 시 자동 실행"**) 토글도 제공합니다.

> **업그레이드 중인가요?** 예전에 훅을 설치했다면 **"Claude Code 훅 설정"** 을 껐다
> 켜세요 — 최신 버전에서 `PostToolUse`, `SubagentStart`, `SubagentStop` 훅이
> 추가되었습니다(작업 중 회전 화살표 유지 + subagent 두 번째 아이콘). 새 훅은 이후에
> 시작한 세션부터 적용됩니다.

**5. 프로젝트마다 tmux 세션 하나**를 각자 자기 터미널 창에 붙여 실행합니다. 등록된
세션이 없으면 패널은 아무 일도 하지 않습니다 — 훅이 첫 상태 파일을 쓰기 전까지 tmux
호출을 **한 번도** 하지 않습니다.

<details>
<summary>훅을 직접 손으로 연결하고 싶다면?</summary>

`wiring.RECOMMENDED_HOOKS`의 훅들을 `~/.claude/settings.json`에 추가하고, 각각을
**절대 경로**로 `<저장소>/bin/cc-navigator-hook`을 가리키게 하세요. 4단계의 설정
대화상자 토글이 정확히 이 작업을 해주므로, 대부분은 직접 할 필요가 없습니다.
</details>

### ⚠️ doctor가 잡으려고 존재하는 tmux 지뢰

tmux 3.0a에서는, `set` 계열(`set`, `setw`, `set-option`, `set-window-option`)의
`~/.tmux.conf` 줄 중 플래그에 **`-g`, `-q`, `-s`가 하나도 없는** 것이 설정 로드
시점에 서버를 조용히 손상시킵니다. 서버는 이후 정상적으로 동작하다가 — 페인을
나열하고 창을 전환하다가 — `send-keys`로 **공백**이 전달되는 순간 **세그폴트**가
나면서, 그 안의 모든 Claude Code 세션을 함께 데려갑니다. 방아쇠는 당신이 입력하는
공백이 든 첫 답장입니다.

`bin/cc-navigator-doctor`는 설정을 임시 서버에 로드하고 공백을 보내 이를 감지하므로,
cc_navigator가 실제 tmux를 건드리기 전에 알 수 있습니다. 해결은 `-g`를 붙이는
것입니다:

```diff
-set mode-keys vi
+setw -g mode-keys vi
```

## 사용 방법

**목록 읽기.** 각 세션은 한 줄입니다: 상태 표시, 제목, 그리고 (선택했을 때) 작업
디렉터리와 마지막 프롬프트.

- 🔴 **빨간 점** — 세션이 당신을 기다림(권한 요청, 질문, 승인할 계획). 제목 표시줄
  배지가 이 개수를 셉니다.
- 🟢 **초록 점** — 세션이 턴을 끝내고 대기 중. 클릭하면 "봤음"(빈 박스)으로 토글되고,
  다시 누르면 돌아옵니다.
- ↻ **회전 화살표** — Claude가 작업 중.
- **겹친 두 아이콘** — 세션에 subagent가 도는 중: 앞은 main 상태, 뒤는 subagent
  spinner. subagent가 있으면서 main이 작업 중이면 앞은 차분한 파란 점이 되고 도는 건
  뒤 도우미뿐입니다.

상단의 *"정렬"* 드롭다운으로 **보기를 전환**합니다:

- **상태별 정렬** — 세 섹션: *입력이 필요한 세션*, *보고 완료*, *작업 중*.
- **그룹별 정렬** — 프로젝트 디렉터리마다 폴더 하나, 각자 상태 카운트를 가짐.

<p align="center">
  <img src="docs/images/screenshot-groups.png" alt="프로젝트별로 그룹화한 cc_navigator" width="360">
</p>

**세션에 대해 동작하기.** 행을 클릭하면 펼쳐집니다:

- 답장 상자에 입력하고 **Enter**를 누르면 그 세션으로 한 줄이 전송됩니다.
- **"해당 세션으로 이동"** 을 클릭하면 그 터미널 창이 앞으로 옵니다(GNOME `Eval`
  필요).

**그룹 정리하기**(그룹 보기에서): 행 오른쪽의 **⠿ 손잡이**를 드래그해 순서를 바꾸거나
다른 그룹 위에 놓고, **연필**을 눌러 그룹 이름을 바꾸고, 쉐브론으로 그룹을 접고,
**"자동 정렬"** 을 누르면 디렉터리 기준으로 다시 그룹화됩니다.

**접기 & 붙이기.** 왼쪽 위 쉐브론 버튼이 패널을 제목 표시줄만 남기고 접습니다. 그
버튼을 **길게 누르면** 붙이기 선택창이 열립니다 — 패널을 아무 화면 가장자리에 딱
붙는 얇은 바로 도킹하면, 아이콘·입력 필요 개수·떼기 버튼만 남습니다. 붙은 상태에서는
**바를 가장자리를 따라 드래그**할 수 있고, 떼기 버튼으로 전체 패널을 복원합니다.

**설정**(⚙ 톱니바퀴)에는 투명도, 배경색, 글꼴 크기, 화면 모서리, 항상 위 / 모든
워크스페이스, 원클릭 자체 업데이트, 그리고 위 설치 단계의 **통합** 토글이 있습니다.

## 프로젝트 구조

```
bin/
  cc-navigator            런처 — 앱을 exec하고 오류를 표면화
  cc-navigator-hook       Claude Code 훅 shim — 모든 것을 삼키고 항상 exit 0
  cc-navigator-doctor     사전 요구사항 점검기
src/ccnav/
  paths.py       상태 디렉터리 (mode 0700)
  hookstate.py   훅 이벤트 → (상태, 이유); 순수 함수
  statestore.py  원자적 쓰기 / read_all / prune — 유일한 파일시스템 소유자
  hook.py        shim의 로직
  proc.py        유일한 서브프로세스 호출 지점; 타임아웃으로 제한
  tmuxctl.py     tmux 질의와 동작
  gnome.py       창을 제목으로 활성화한 뒤 xprop으로 실제 여부를 증명
  model.py       상태 파일과 살아 있는 tmux 페인을 조인 → 행; 순수 함수
  ui.py          오버레이 창; 위젯 위의 포매팅은 순수 함수
  wiring.py      런처 / 자동 실행 / settings.json 훅 병합 (통합 토글)
  updater.py     안전한 fast-forward 자체 업데이트
  doctor.py      사전 요구사항 점검(세그폴트 재현 포함)
  app.py         배선 — GTK 메인 스레드에서 블로킹 작업이 절대 돌지 않음
```

설계·계획·작업별 엔지니어링 기록은 [`docs/`](docs/) 아래에 있습니다 —
[`docs/superpowers/sdd/implementation-log.md`](docs/superpowers/sdd/implementation-log.md)에서
시작하세요.

## 상태

전체 기능이 구현되었고 테스트 스위트가 통과합니다(**445개 테스트**). 각 작업은
**뮤테이션 테스트**로 검증하고 — 구현을 일부러 N가지로 망가뜨려 각 손상이 이름 붙은
테스트를 반드시 실패시켜야 합니다 — 가장 위험한 변경은 병합 전에 적대적 다중 에이전트
리뷰를 거칩니다.
